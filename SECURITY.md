# Security Policy

## Reporting a vulnerability
- Email: security@sans-serif-sentiments.example (replace with a working alias if different).
- Please include: affected area (doc/site/script), reproduction steps, impact, and any logs you can share safely.
- Do not open public issues for potential security problems.

## Scope
- Handbook docs and the Docusaurus site.
- Included scripts/snippets/experiments in this repo.

## Expectations
- We aim to acknowledge reports within 3 business days.
- Please allow time for assessment and remediation before disclosure.

## Safe handling
- Do not send secrets or customer data.
- If your report involves sensitive data, describe the scenario; do not paste the data itself.
